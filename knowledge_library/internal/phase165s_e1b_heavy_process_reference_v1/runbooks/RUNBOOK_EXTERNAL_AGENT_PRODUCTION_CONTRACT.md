# External Agent Production Contract

## Purpose
Use when Builder is asked to produce a child agent, automation, product engine or digital asset.

## Acceptance
External agent production is accepted only when Builder:
1. defines child goal and boundaries;
2. grows/builds the child agent or organ;
3. runs it in sandbox;
4. validates behavior;
5. records proof/report;
6. records what was learned;
7. does not make Codex the normal factory.

## Proof package
- child spec;
- generated files;
- sandbox run log;
- validator output;
- report;
- limitations and next action.

## Do not
- deliver a proposal as if it were a working agent;
- use external LLM as core identity;
- install unreviewed frameworks;
- skip sandbox run.
