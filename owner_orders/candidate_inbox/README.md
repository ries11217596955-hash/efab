# Owner Candidate Inbox

Put candidate files here using this pattern:

*.candidate.json

Important:
A candidate is not an accepted atom.

Candidate flow:
1. owner supplies candidate;
2. Builder checks shape and provenance;
3. Builder sends it to quarantine/sandbox;
4. Builder validates;
5. Builder creates proof;
6. Builder promotes or rejects.

No direct self-modification happens from this inbox.
