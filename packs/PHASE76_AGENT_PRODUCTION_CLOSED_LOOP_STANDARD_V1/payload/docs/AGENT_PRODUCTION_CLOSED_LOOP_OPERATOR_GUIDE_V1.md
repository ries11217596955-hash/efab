# Руководство оператора: закрытый цикл производства агента v1

## Что такое закрытый цикл

Закрытый цикл означает, что Builder не просто создаёт файлы агента. Он проходит весь путь от формальной программы до проверенного результата: собирает пакет, проверяет локальный runtime, регистрирует агента, запускает GitHub Actions, скачивает артефакт, валидирует его, пишет proof/report и только потом возвращает очередь в `NONE`.

## Почему созданный агент не принимает сам себя

Созданный агент является объектом проверки, а не судьёй. Если агент сам объявит себя принятым, фабрика потеряет независимую приёмку: ошибка в агенте сможет выглядеть как успех. Поэтому агент может только выполнить свою работу и выдать артефакт.

## Почему приёмку делает Builder

Builder владеет состоянием репозитория, очередью задач, каталогом агентов, proof/report и проверкой GitHub run. Он видит весь контур: что было запущено, какой run завершился, какой артефакт получен, что сказал validator, и был ли каталог переведён в `ACCEPTED`.

## Где смотреть proof и report

Основной proof создаётся здесь:

`proofs/AGENT_PRODUCTION_CLOSED_LOOP_STANDARD_V1.json`

Отчёт создаётся здесь:

`reports/external_agent_production/AGENT_PRODUCTION_CLOSED_LOOP_STANDARD_V1_REPORT.json`

Для доказанной базы стандарт ссылается на:

`proofs/RUNBOOK_EXECUTOR_AGENT_GITHUB_ACTION_ACCEPTANCE_V1.json`

## Что владелец должен увидеть в GitHub Actions

Владелец должен видеть workflow run, который был запущен Builder-ом, завершился с conclusion `success` и опубликовал ожидаемый artifact. После этого Builder должен скачать artifact и подтвердить `validation_status = PASS`.

## Что считается настоящим ACCEPTED

Настоящий `ACCEPTED` существует только когда одновременно выполнены условия:

- GitHub run завершился с `success`;
- artifact скачан и проверен;
- artifact validation равен `PASS`;
- agent catalog показывает статус `ACCEPTED`;
- proof и report записаны;
- `TASK_QUEUE.active_task_id` возвращён в `NONE`.

Если хотя бы одного пункта нет, это не принятие, а промежуточное состояние или BLOCKED.
