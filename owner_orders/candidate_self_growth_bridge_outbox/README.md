# Candidate Self-Growth Bridge Outbox

This folder stores generated bridge-task previews for owner-supplied candidates.

Important:
This is not a promotion folder.
This does not accept atoms.
This does not mutate Builder.

Purpose:
Convert a valid owner candidate into a task-shaped material package for the existing Builder self-growth loop.

Correct flow:
candidate -> inbox -> quarantine/sandbox gate -> self-growth bridge -> existing Builder organs -> sandbox -> validation -> proof -> accept/reject
