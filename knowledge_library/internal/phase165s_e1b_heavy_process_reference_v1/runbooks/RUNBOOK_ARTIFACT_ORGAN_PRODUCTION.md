# Artifact Organ Production

## Purpose
Use when Builder must produce docx, spreadsheet, slides, PDF or other delivery files.

## Process
1. Identify requested artifact and format.
2. Check existing organ/tooling and current system rules.
3. For spreadsheets use spreadsheet-specific pipeline; do not use document/PDF tools unless requested.
4. For docs/slides/PDF read relevant skill/instructions before creating.
5. Create artifact in sandbox or repo output path.
6. Validate file exists, size > 0 and content sanity.
7. Provide link or repo path plus proof.
8. Do not claim artifact is delivered without actual file link/proof.

## Failure modes
- file path invented;
- wrong format;
- no validation;
- claim completed without artifact;
- use of forbidden tooling.
