# PHASE164C Owner Candidate Inbox Intake

Purpose:
Create controlled inbox for owner-supplied ready candidates.

This is the second mechanism:
- external/owner candidate enters as material;
- Builder checks shape and provenance;
- Builder does not accept it directly;
- Builder prepares quarantine/sandbox review.

Candidate flow:
candidate -> intake -> quarantine -> sandbox -> validation -> proof -> promote/reject -> memory update

This phase does not:
- mutate accepted core;
- mutate route lock;
- execute Codex;
- promote any candidate.
